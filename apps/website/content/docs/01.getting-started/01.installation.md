# Getting Started

Inkline is **write-once, compile-everywhere**. You author a component once in a
`.ink.tsx` file, and the compiler emits idiomatic output for React, Vue 3,
Svelte 5, Solid, Angular, Qwik, and Astro. The generated components carry no
runtime dependency on Inkline.

This is deliberately not a per-framework plugin you install into one app — it is
a compiler you point at your source.

## Install

Add the compiler to your project:

```bash [npm]
npm install @inkline/compiler
```

```bash [pnpm]
pnpm add @inkline/compiler
```

TypeScript 5.4 or newer is required as a peer dependency. Authoring primitives
(`createSignal`, `createMemo`, `createEffect`, `defineComponent`, `Slot`) come
from `@inkline/core`.

> Prefer a guided setup? `npx inkline init --compiler` detects your package
> manager, framework and bundler, scaffolds an `inkline.config.ts`, and drops in
> an example component.

## Author a component

Write your component once in `.ink.tsx`:

```tsx [src/components/Counter.ink.tsx]
import { createSignal, createMemo, defineComponent } from "@inkline/core";

export default defineComponent(() => {
  const [count, setCount] = createSignal(0);
  const doubled = createMemo(() => count() * 2);

  return (
    <div>
      <p>{count()}</p>
      <p>{doubled()}</p>
      <button onClick={() => setCount(count() + 1)}>+1</button>
    </div>
  );
});
```

## Compile everywhere

Point the compiler at your source and choose your targets:

```bash
npx inkline compile src/components/Counter.ink.tsx \
  --target react,vue,svelte,solid,angular,qwik,astro \
  --out-dir dist
```

A single source produces one idiomatic component per framework:

```
Counter.ink.tsx  ──▶  react/Counter.tsx
                 ──▶  vue/Counter.vue
                 ──▶  svelte/Counter.svelte
                 ──▶  solid/Counter.tsx
                 ──▶  angular/Counter.component.ts
                 ──▶  qwik/Counter.tsx
                 ──▶  astro/Counter.astro
```

For repeatable builds, declare your targets in `inkline.config.ts` instead of
flags, then run `inkline compile` with no arguments. Use `inkline check` to run
compiler diagnostics without writing any output.
